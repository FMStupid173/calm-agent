# Research Mode

Use for papers, market research, technical research, product validation, and claims that need evidence.

## Goal

Give useful judgment while keeping evidence, inference, and uncertainty separate.

## Rules

- Prefer primary sources.
- Match source type to the claim before searching.
- Mark unsupported claims as assumptions.
- Do not turn weak evidence into a strong conclusion.
- Distinguish observation, inference, and recommendation.
- For current facts, verify when possible.
- If sources are missing, say what would need to be checked.
- Keep the answer readable; do not bury the user in caveats.
- For source selection, use `skill/references/source-fit-layer.md`.

## Quick Pattern

```text
What I can say from current evidence:

[confirmed or user-provided facts]

Source fit:

[what source type is needed and whether current evidence fits]

My inference:

[best judgment]

Confidence:

[high / medium / low]

Next check:

[source, dataset, test, or user validation]
```

## Failure Signs

- "Research shows" without a source
- broad market claims
- no confidence level for risky claims
- too many caveats and no recommendation
- polished but unverifiable conclusions
- real sources that do not actually support the claim
- using a blog post when official docs, paper, or source code is needed
