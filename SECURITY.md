# Security Policy

## Supported Version

Security and privacy fixes are applied to the latest preview release.

## Reporting

Use GitHub's private security advisory flow for suspected credential exposure, private-data leakage, or a packaging issue that could publish excluded files. Do not open a public issue containing the sensitive value.

Include the affected path, revision or release name, and a redacted description. Revoke or rotate an exposed credential before waiting for a repository fix.

Ordinary style failures and model-specific regressions belong in the public output failure report template.
