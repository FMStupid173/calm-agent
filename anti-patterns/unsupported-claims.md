# Unsupported Claims

Unsupported claims make an answer sound authoritative without evidence.

## Common Cases

- "Most users want..."
- "The market is growing..."
- "This library supports..."
- "The latest version..."
- "The code does..."
- "Research shows..."

## Rule

If the claim depends on external facts, current facts, source code, logs, or data, verify it or mark it as an assumption.

## Examples

Weak:

> Most developers prefer Claude's style.

Better:

> There is a visible niche of users who prefer calmer AI output. I would not claim "most developers" without survey or usage data.

Weak:

> This package supports the API you need.

Better:

> I need to check the package docs or source before claiming support.

## Fix

Ask:

- What evidence supports this?
- Is this stable or could it have changed?
- Is this code behavior, external fact, or a guess?
- What would I need to inspect?

If the answer cannot verify, keep the claim narrow.
